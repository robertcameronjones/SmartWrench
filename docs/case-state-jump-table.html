Case state jump table — as implemented

Source of truth: 11Labs/src/guidepoint/case/_reducer.py on branch v2. Every row maps to a single branch in the reducer. The reducer is total: any (state, signal) pair not listed in a state's block is handled by the cross-cutting rules below, or by the implicit rule "anything else returns ignored:<why>, no state change, no actions".

Why some SHOWED rows are marked "defensive"

The reducer routes SHOWED calls through the FEEDBACK call stage (_stage_for_state(SHOWED) == FEEDBACK) and accepts feedback / inconclusive CallEnded outcomes. But no v2 reducer branch ever emits PlaceCall(stage=FEEDBACK), so those branches only fire if something outside the v2 flow (legacy code, test setup, or operator action) places a feedback call. The happy path from SHOWED is VehicleExitedDealer going straight to COMPLETED.

Terminal states

StateReached via
DECLINEDCONTACTING_CUSTOMER + CallEnded(declined).
CANCELLEDINITIAL_REMINDER_SENT, FINAL_REMINDER_SENT, or RESCHEDULING + CallEnded(cancelled).
ABANDONEDInconclusive call at attempt_count == 2 in CONTACTING_CUSTOMER or RESCHEDULING.
NO_SHOWFINAL_REMINDER_SENT + EndOfBusinessDayReached (no geofence-in by close of business).
RESCHEDULE_FAILEDCustomer requested a second reschedule at either reminder (reschedule_count == 1 when CallEnded(rescheduled) arrives).
OPTED_OUTCross-cutting CustomerOptedOut, or CallEnded(opted_out) from any non-terminal state.
COMPLETEDHappy path: SHOWED + VehicleExitedDealer. The audit trail gets both case.geofence.exited and case.closed.service_event_complete. Also reachable via the defensive feedback-call branches above.
BOOKEDLegacy v1 enum value. No v2 reducer branch ever lands here.

States in the enum the v2 reducer never lands in

  • SLOT_PROPOSED, SLOT_PICKED — legacy v1 enum values. _on_decided_outcome and _stage_for_state still handle them identically to CONTACTING_CUSTOMER (so migrated v1 cases keep working), but no v2 reducer branch transitions into them.
  • AWAITING_FEEDBACK — legacy. The feedback handlers in _on_decided_outcome and _on_incomplete_outcome accept it alongside SHOWED; no v2 reducer branch transitions into it.
  • BOOKED — legacy terminal. Replaced by the CONFIRMING_WITH_DEALER → INITIAL_REMINDER_DUE / FINAL_REMINDER_DUE path. No v2 reducer branch lands here.

  • Every CaseDecision carries a reason string. No-op decisions use reason="ignored:<why>" so the driver can log them at debug level without losing the audit trail.
  • The reducer is pure: it returns actions; the driver applies them. Actions are: PlaceCall, RequestDealerSlots, RequestDealerConfirmation, ScheduleTimer, CancelTimer, RecordEvent.
  • Of those, only PlaceCall ever causes an LLM call downstream (via the channel's CallSession.start()). Every other action is bookkeeping / I/O against the dealer port or timer wheel. Sliders (geofence, EoD, business hours) never produce PlaceCall.
